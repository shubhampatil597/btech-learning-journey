# Day 5 – File System, Multitasking & Productivity

## 1. File System

A file system is a method used by the operating system to store, organize, and manage data efficiently.

Without a file system, data would be unorganized and difficult to access.

---

## 2. How Data is Stored

Data is stored in storage devices (HDD/SSD) in the form of files and folders.

The operating system keeps track of:
- File name
- File location
- File size
- File permissions

---

## 3. File

A file is a unit of data.

Examples:
- document.pdf
- image.jpg
- video.mp4

Each file has:
- Name
- Extension
- Size
- Path

---

## 4. Folder (Directory)

A folder is a container that holds files and other folders.

It helps in organizing data in a structured way.

---

## 5. Path

Path represents the exact location of a file.

Example:
C:/Users/Shubham/Documents/Notes/file.pdf

---

## 6. Types of Path

### Absolute Path
- Full location from root directory
- Example: C:/Users/Shubham/Documents/file.pdf

### Relative Path
- Location from current directory
- Example: Notes/file.pdf

---

## 7. Multitasking

Multitasking means running multiple applications at the same time.

Example:
- Browser
- Music
- Notes

---

## 8. How Multitasking Works

- CPU switches between tasks quickly (context switching)
- RAM stores active programs
- OS manages all tasks

This creates the illusion that everything runs at the same time.

---

## 9. Why System Slows Down

- Low RAM
- Too many applications open
- HDD (slower than SSD)
- Background processes

---

## 10. Productivity System

Organizing files and folders improves efficiency and saves time.

### Recommended Folder Structure

---

## 11. File Naming Rules

Bad:
- file1.pdf
- notes_final_latest.pdf

Good:
- OS-notes-day4.pdf
- HTML-project-landingpage.pdf

---

## 12. Key Learnings

- File system organizes data efficiently
- Path helps locate files
- Multitasking uses CPU and RAM
- Clean system improves performance
- Organization is important for productivity
